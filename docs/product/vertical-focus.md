# Vertical Focus

## 目标

在“情感沟通”和“职场表达”两个方向里做聚焦，避免产品长期停留在泛场景助手。

## 当前判断

基于当前仓库的页面、模板和商业化潜力，建议：

- 短期产品体验继续同时覆盖两类场景
- 中期优先把 `职场表达` 作为收费验证主方向
- 保留 `情感沟通` 作为自然增长和高情绪需求入口

原因不是哪个方向“更正确”，而是它们在增长和变现上承担的角色不同。

## 为什么不是只做单一方向

当前代码和文案已经天然覆盖了两端：

- 首页示例同时包含同事、朋友、伴侣：`high-eq-front/client/src/pages/Home.tsx`
- 生成页角色同时覆盖朋友、家人、领导、客户、伴侣：`high-eq-front/client/src/pages/ReplyApp.tsx`
- 新增模板数据也分成了两个分组：`high-eq-front/client/src/data/scenarioTemplates.ts`

这说明当前产品最适合先保留双入口，再用埋点看谁更强。

## 推荐聚焦策略

### 增长入口：情感沟通

适合做自然传播的原因：

- 痛点直接
- 情绪浓度高
- 更容易在社交媒体形成共鸣
- 更容易吸引第一次尝试

适合内容主题：

- 伴侣冷战怎么破冰
- 怎么表达边界但不伤人
- 怎么礼貌拒绝朋友
- 怎么安抚焦虑情绪

### 付费入口：职场表达

适合做收费验证的原因：

- 用户更愿意为效率和结果付费
- 需求更容易重复出现
- 更容易解释 Pro 价值
- 更容易后续延展到 ToB

适合重点强化的场景：

- 回复领导
- 催同事进度
- 给客户解释报价
- 会后跟进
- 拒绝不合理需求
- 沟通延期风险

## 建议阶段

### 阶段 1

保留双分组模板，观察：

- 模板点击率
- 首次生成成功率
- 收藏率
- 7 日回访率

### 阶段 2

如果职场场景在以下指标更强，就把 Pro 更偏向职场：

- 重复使用频次
- 升级 CTA 点击率
- 收藏率
- 二次改写使用率

### 阶段 3

只有当某一端明显强于另一端时，再决定：

- 独立 landing page
- 独立品牌文案
- 独立定价策略

## 建议埋点

必须区分：

- `template_category = relationship`
- `template_category = workplace`

建议重点看 6 个指标：

1. 新用户模板点击率
2. 首次生成完成率
3. 复制率
4. 收藏率
5. 次日回访率
6. Pro CTA 点击率

## 当前推荐

如果你现在就要选一个更适合赚钱的方向，建议先押：

- `职场高情商表达助手`

如果你现在更看重自然增长和传播内容，建议先押：

- `情感与关系沟通助手`

## 最终建议

HighEQ 当前最合理的打法不是二选一，而是：

1. 用情感沟通模板做增长入口
2. 用职场表达权益做 Pro 验证
3. 用埋点数据决定后续品牌是否拆分
